# Agent Output Directory

This directory contains all artifacts generated by specialized AI sub-agents during the Software Development Lifecycle (SDLC).

## Quick Reference

| What You Need | Where to Find It |
|---------------|------------------|
| Validated product requirements | `briefs/validated/` |
| Technical specifications | `specs/` (frontend, backend, database, etc.) |
| Implementation plans | `../plans/` (one level up) |
| Interface contracts | `contracts/` (API, services, components, schemas) |
| Test strategies | `tests/specs/` |
| Test results | `tests/reports/` (integration, e2e, security, etc.) |
| Code review findings | `reviews/` |
| Generated documentation | `docs/` (API, architecture, deployment, guides) |

## Directory Overview

```
output/
├── briefs/              # Product brief validation (Phase 1)
├── specs/               # Technical specifications (Phase 2)
├── contracts/           # Interface contracts (Phase 4)
├── tests/               # Test specs and reports (Phase 5, 8)
├── reviews/             # Code review reports (Phase 7)
└── docs/                # Generated documentation (Phase 9)
```

## When Files Appear Here

1. **After validating a product brief** → Check `briefs/validated/`
2. **After creating technical specs** → Check `specs/frontend/`, `specs/backend/`, etc.
3. **After contract generation** → Check `contracts/api/`, `contracts/services/`, etc.
4. **After running tests** → Check `tests/reports/<test-type>/`
5. **After code review** → Check `reviews/`
6. **After documentation generation** → Check `docs/`

## File Naming Pattern

Most files follow this pattern:
```
<type>-<feature-name>-<YYYY-MM-DD>.md
```

Examples:
- `briefs/validated/messenger-app-validated-brief-2026-02-04.md`
- `specs/backend/backend-spec-message-api-2026-02-04.md`
- `tests/reports/integration/integration-report-2026-02-04.md`

## Metadata Headers

All generated files include metadata:

```markdown
---
generated_by: <agent-name>
generation_date: YYYY-MM-DD HH:MM:SS
source_input: <input-file-path>
version: X.Y.Z
tech_stack: <detected-technologies>
status: draft | review | approved | deprecated
---
```

## Finding Specific Outputs

**Latest test results:**
```bash
ls -t tests/reports/integration/ | head -1
```

**All specs for a feature:**
```bash
find specs/ -name "*message*"
```

**Recent code reviews:**
```bash
ls -t reviews/ | head -5
```

## For Developers

- **Don't manually edit** generated files - they may be overwritten
- **Reference these files** in your implementation
- **Track changes** in version control to see evolution
- **Archive old reports** periodically (see OUTPUT_STRUCTURE.md)

## For Agents

When writing outputs:
1. Use paths defined in `../.claude/OUTPUT_STRUCTURE.md`
2. Include metadata header
3. Follow naming conventions
4. Create directories if they don't exist
